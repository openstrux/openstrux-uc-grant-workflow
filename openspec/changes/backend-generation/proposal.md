## Why

The grant workflow repository is initialized with a frontend, contract stubs, and tests тАФ but no backend implementation. The backend must be generated by either of two paths (direct or Openstrux-assisted) against the same baseline. This change defines exactly what must be built тАФ it is the task that both paths execute.

## What Changes

### Contract-first baseline

The baseline contains typed stubs that define the contract between front-end and back-end. Tests import only from these contract surfaces. Internal file structure within packages is free.

### Prisma schema
- Complete `prisma/schema.prisma` with all P0-P2 models: `Submission`, `ProposalVersion`, `ApplicantIdentity`, `BlindedPacket`, `EligibilityRecord`, `AuditEvent`, `Call`
- Relations and indexes as required by the domain model

### Domain package (`packages/domain/`)
- `packages/domain/src/schemas/index.ts` тАФ Zod schemas for all entities + API request/response shapes (single source of truth; all types derived via `z.infer<>`)

### Policy package (`packages/policies/`)
- `packages/policies/src/index.ts` тАФ barrel exporting pure business-logic functions:
  - `evaluateEligibility` тАФ rule evaluation against active rule set
  - `createBlindedPacket` тАФ strip identity fields from proposal content
  - `isValidTransition` / `getNextStatus` тАФ workflow state machine
- Internal file structure is free as long as the barrel resolves

### Service layer (`src/server/services/`)
- `submissionService.ts` тАФ `submitProposal`, `listSubmissions`, `getSubmission`
- `eligibilityService.ts` тАФ `runEligibilityCheck`

### Auth (DAL)
- `src/lib/dal.ts` тАФ `verifySession(req)` тЖТ `Principal | null` (dev-mode `X-Role`/`X-User-Id` headers for P0-P2)

### API routes (`src/app/api/`)
- `src/app/api/intake/route.ts` тАФ POST handler (replace stub), validates with `IntakeRequestSchema`
- `src/app/api/eligibility/route.ts` тАФ POST handler (replace stub), validates with `EligibilityRequestSchema`

## Acceptance criteria

All tests in `tests/unit/` and `tests/integration/` pass.
