## Why

The grant workflow repository has a complete frontend, contract stubs, and test suite — but no backend implementation. The backend must be generated by either of two paths (direct or Openstrux-assisted) against the same baseline to produce a working, fully-tested application. Additionally, a post-generation gap review (GAP-001) identified that the `firstTimeApplicantInProgramme` eligibility check was missing from both the MVP rule set and the test baseline, requiring a coordinated spec + test correction before the next benchmark run.

## What Changes

- **New: Prisma schema** — `prisma/schema.prisma` with all P0–P2 models and cascade constraints
- **New: Domain schemas** — `src/domain/schemas/index.ts` Zod schemas replacing stubs
- **New: Policy functions** — `evaluateEligibility` (6-rule set), `createBlindedPacket`, `isValidTransition`, `getNextStatus` in `src/policies/index.ts`
- **New: Service layer** — `submitProposal`, `listSubmissions`, `getSubmission` in `submissionService.ts`; `runEligibilityCheck` in `eligibilityService.ts`
- **New: API routes** — `POST /api/intake` and `POST /api/eligibility` replacing stubs
- **New: Seed** — `prisma/seeds/seed.ts` with one user per role + default `Call` record (6 enabled eligibility checks)
- **Updated: test baseline** — `tests/unit/eligibility.test.ts`, `tests/integration-mock/eligibility.test.ts`, and `tests/fixtures/eligibility/*.json` updated to include `firstTimeApplicantInProgramme` in active rules (GAP-001 fix)

## Capabilities

### New Capabilities

- `eligibility-evaluation`: Pure rule evaluation of `EligibilityInputs` against a call's active rule set; 6 checks including `firstTimeApplicantInProgramme`; result includes `status`, `failureReasons`, `inputs`, `activeRules`
- `proposal-intake`: End-to-end proposal submission — Submission + ProposalVersion + BlindedPacket + AuditEvent created atomically
- `workflow-state-machine`: `isValidTransition` / `getNextStatus` enforcing allowed status transitions per spec

### Modified Capabilities

- `eligibility-evaluation`: GAP-001 — `firstTimeApplicantInProgramme` added to `MVP_DEFAULT_RULES` and `Call.enabledEligibilityChecks` seed; test baseline updated to match

## Impact

All files under `src/policies/`, `src/server/services/`, `src/app/api/intake/`, `src/app/api/eligibility/`, `prisma/`, and test baseline files `tests/unit/eligibility.test.ts`, `tests/integration-mock/eligibility.test.ts`, `tests/fixtures/eligibility/`.
