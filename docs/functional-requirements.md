# Functional Requirements

The authoritative functional requirements are in `openspec/specs/`:

- `openspec/specs/domain-model.md` — P0 canonical model (FR-P0-001–005): entities, identity separation, workflow states, role visibility, explicit policies
- `openspec/specs/workflow-states.md` — P0/P2 state machine and access rules per state
- `openspec/specs/access-policies.md` — P0 role definitions, access policies, enforcement points
- `openspec/specs/mvp-profile.md` — P0–P2 active phases, enabled eligibility checks
- `openspec/changes/backend/specs/eligibility-evaluation/spec.md` — P2 eligibility rules (FR-P2-001–008)
- `openspec/changes/backend/specs/proposal-intake/spec.md` — P1 intake and identity separation (FR-P1-001–009)
- `openspec/changes/backend/specs/workflow-state-machine/spec.md` — P0/P2 state transition enforcement

P3–P6 are deferred. See `docs/scope.md`.
