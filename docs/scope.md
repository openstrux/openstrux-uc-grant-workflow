# Scope

## In scope (v0.6.0 demo)

- P0: Typed canonical model for identity, proposal, workflow states, access roles, policy checks, audit events
- P1: Submission intake with lightweight UI, identity/proposal data separation, blinded packet generation
- P2: First-stage eligibility gate using explicit boolean/numeric inputs only
- Side-by-side generation paths (direct TS vs Openstrux-assisted) over the same baseline
- Benchmark-ready structure for comparison

## In scope (full MVP, future phases)

- P3: Review access, confidentiality/CoI acknowledgement, manual scoring
- P4: Clarification and revision workflow
- P5: Independent validation
- P6: Audit trail, retention/deletion lifecycle

## Out of scope

- Automated proposal reading or semantic evaluation
- English-language detection or content classification
- Automated project selection or funding recommendation
- Grant execution, milestone payment, post-award administration
- Full production-grade legal compliance package
- Multi-tenant enterprise scaling beyond MVP needs
